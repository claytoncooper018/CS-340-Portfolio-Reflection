# CS 340 – Database Systems Portfolio Reflection

## Overview
My work from CS 340, including the dashboard application and accompanying documentation, is included in this repository. This project showcases my skills to create a functional dashboard utilizing a Python-based CRUD module and to design and interface with a MongoDB database.

---

## Writing Maintainable, Readable, and Adaptable Programs
I concentrated on modular design, a clear structure, and consistent naming standards to produce legible and maintainable applications. To manage database activities in Project One, I created a reusable CRUD Python program. Because all database interactions were concentrated in one place, my code became more structured and simpler to debug.

I was able to effectively connect the dashboard widgets to the database without rewriting database logic by using Project Two's CRUD module. Because future modifications to database operations may be done in one location without impacting the entire program, this strategy increased adaptability.

Future applications that need database interaction, like web apps, data analysis tools, or automation systems, might make use of this CRUD module.

---

## Problem-Solving Approach as a Computer Scientist
I divided the needs for this project into manageable chunks, like database setup, query development, and dashboard features. I concentrated on comprehending the data needs for the Grazioso Salvare scenario before creating queries to enable the dashboard capabilities.

Because it required merging several components, such as the database, backend logic, and user interface, this method was different from earlier assignments. To guarantee correctness and functioning, I tested every component as I constructed it using an iterative technique.

In order to create databases that successfully satisfy client needs, I would keep utilizing this methodical approach in conjunction with meticulous planning and testing in subsequent projects.
